# SECTION 1: GEO KNOWLEDGE COMPENDIUM
## Audit Intelligence

---

## A. Strategic Reframe: The "Closer" Module

The old paradigm of SEO audits was a 50-page PDF of technical errors (404s, redirect chains) that clients never read and rarely fixed. In the Agentic Era, an audit is not a list of problems; it is a **Strategic Gap Analysis.**

Module 7 is the "Closer." It bridges the gap between "I think we need SEO" and "Here is the exact revenue we are losing." It creates the demand for Modules 1-6 by mathematically proving where the client is losing to competitors.

---

## B. The 5 Core Pains (Audit Deep Dive)

### Pain 1: Audit Paralysis

**Symptom:** "We got a 100-page audit from Semrush, but we don't know where to start. It's just a list of warnings."

**Root Cause:** Data without prioritization. Technical tools flag everything equally (a missing alt tag vs. a blocked money page).

**Strategic Implication:** The audit must separate **Revenue Blockers** (Critical) from **Housekeeping** (Nice to have). We use the MMA scoring model (Money, Movement, Authority) to prioritize fixes.

---

### Pain 2: Competitor Blindspot

**Symptom:** "We have better content, but they rank #1."

**Root Cause:** Analyzing competitors at the keyword level instead of the structure level. You missed that their H2 structure satisfies "Goal Completion" better than yours, or they have a schema type you lack.

**Strategic Implication:** We must move from "Keyword Gap" to "Structural Gap" analysis using the Header Hierarchy Extraction pipeline.

---

### Pain 3: Gap-to-Action Disconnect

**Symptom:** "The audit was done 3 months ago, but traffic hasn't changed."

**Root Cause:** The audit was a "deliverable," not a "directive." It didn't feed into a creation engine.

**Strategic Implication:** Module 7 must directly output **Content Briefs for Module 3** (Writer). The audit is the input for the production line.

---

### Pain 4: Surface-Level Metrics

**Symptom:** "Our health score is 98%, but we have no leads."

**Root Cause:** Optimizing for tools, not entities. Your technical SEO is perfect, but your "Entity Strength" and "Topical Authority" are weak.

**Strategic Implication:** The audit must evaluate **AI Readiness** (Schema, Entities, Citations) and **BOFU Coverage**, not just crawl errors.

---

### Pain 5: Client Communication Gap

**Symptom:** "The client doesn't understand why 'canonical tags' matter."

**Root Cause:** Speaking "Technician" instead of "Executive."

**Strategic Implication:** Translate every finding into **Revenue Opportunity**. "Fixing this tag" becomes "Unlocking visibility for your $5k product."

# SECTION 3: AGENTIC WORKFLOW ARCHITECTURE
## Based on Vincent Terrasi's Agentic SEO Framework

This module utilizes a multi-agent system to automate the heavy lifting of data collection and analysis, allowing the strategist to focus on insights.

---

## 1. Data Collection Agents (The "Scouts")

### Crawler Agent
**Tool:** Firecrawl / Screaming Frog  
**Function:** Systematically analyzes the client site and competitor sites  
**Extracts:** HTML, metadata, schema, internal links, content  
**Output:** `crawl_data.json`

### SERP Agent
**Tool:** DataForSEO / SerpApi  
**Function:** Monitors search results for target keywords  
**Detects:** AI Overviews, Featured Snippets, ranking shifts  
**Output:** `serp_data.json`

### Metrics Agent
**Tool:** Ahrefs API / Semrush API  
**Function:** Pulls competitive intelligence data  
**Collects:** Backlink data, domain authority, keyword volumes, top pages  
**Output:** `metrics_data.json`

---

## 2. Analysis Agents (The "Detectives")

### Technical Analyst
**Function:** Reviews crawl data for errors  
**Identifies:** 404s, redirect chains, orphan pages, duplicate content  
**Reference:** Matches against KB-TECHNICAL-GEO-FOUNDATIONS (Module 6)  
**Output:** `technical_issues.yaml`

### Content Gap Analyst
**Function:** Compares Client Content vs. Competitor Content  
**Identifies:** Missing topics, entities, keywords, structural gaps  
**Method:** Semantic analysis, entity extraction, header comparison  
**Output:** `content_gaps.yaml`

### Pattern Recognition Agent
**Function:** Looks for structural anomalies  
**Question:** "Why is this competitor ranking with 500 words when we have 2,000?"  
**Answer:** User Intent / Goal Completion / Structure superiority  
**Output:** `pattern_insights.yaml`

---

## 3. Recommendation Agents (The "Strategists")

### Prioritization Agent
**Function:** Scores every finding using MMA Framework  
**Criteria:**
- **Money:** Revenue impact potential
- **Movement:** Likelihood of ranking improvement
- **Authority:** Contribution to topical coverage  
**Output:** Sorted list from "revenue critical" to "minor"

### Strategy Agent
**Function:** Maps findings to specific Modules  
**Logic:**
- Content gap → Module 3 (Writer)
- Technical error → Module 6 (Orchestrator)
- Video opportunity → Module 4 (YouTube)
- Lead magnet opportunity → Module 5 (Fast Track)  
**Output:** `module_recommendations.yaml`

---

## 4. Implementation Agents (The "Builders")

### Brief Generator
**Function:** Takes Gap Analysis and creates Module 3-ready content briefs  
**Input:** `content_gaps.yaml` + `pattern_insights.yaml`  
**Output:** `content_briefs/` directory with individual brief files

### Schema Architect
**Function:** Auto-generates JSON-LD code to fix schema errors  
**Reference:** Module 6 schema templates  
**Output:** `schema_fixes/` directory with ready-to-deploy code

# SECTION 4: COMPETITIVE CONTENT ANALYSIS PIPELINE
## The Core Engine of "Mode B: Ranking Program"

---

## Phase 1: SERP Extraction

**Objective:** Get the raw data of what Google/AI currently rewards.

**Actions:**
1. Input Target Keyword
2. Agent: Scrape Top 10 organic results
3. Extract: URL, Page Title, Meta Description, H1, Word Count, Schema Types, Media Count
4. AI Check: Does an AI Overview appear? (Yes/No)

**Output Template:**
```yaml
serp_extraction:
  keyword: "[Target Keyword]"
  search_date: "2026-01-23"
  ai_overview_present: true/false
  
  results:
    - rank: 1
      url: "https://competitor1.com/article"
      title: "Complete Guide to X"
      meta_description: "..."
      h1: "The Complete Guide to X in 2026"
      word_count: 3450
      schema_types: ["Article", "FAQ", "HowTo"]
      media_count:
        images: 12
        videos: 1
        tables: 3
      
    # ... repeat for positions 2-10
```

---

## Phase 2: Header Hierarchy Extraction (The "Skeleton")

**Objective:** Understand the logic of the winning content.

**Actions:**
1. Extract H2s and H3s from the Top 3 competitors
2. Map the Flow:
   - Competitor A: Problem → Solution → Features → Pricing
   - Competitor B: Definition → History → Types → Examples
3. Identify the "Consensus Structure": What sections do all winners share? (Table stakes)
4. Identify the "Missing Angle": What is no one talking about? (Your opportunity)

**Output Template:**
```yaml
header_analysis:
  competitor_1:
    url: "https://competitor1.com/article"
    h1: "Complete Guide to X"
    h2_sections:
      - "What is X?"
      - "Why X Matters"
      - "How to Implement X"
      - "X Best Practices"
      - "Common Mistakes"
    h3_under_h2:
      "What is X?":
        - "Definition"
        - "History"
        - "Types"
    total_headings: 24
    
  consensus_headers:  # All Top 3 have these
    - "What is X"
    - "Benefits"
    - "How to"
    - "Best Practices"
    
  missing_opportunities:  # None of Top 3 cover
    - "X for [Specific Persona]"
    - "X + AI Integration"
    - "Common X Failures"
```

---

**Objective:** Visualize the battlefield.

**Actions:**

- **Media:** Do they use video?
- **Freshness:** When was it last updated?

# SECTION 6: KNOWLEDGE BLOCKS

## KB-COMPETITIVE-CONTENT-ANALYSIS-v1.0

### 1. The "Goal Completion" Metric

Google and AI agents prioritize **Goal Completion** over Time on Site.

- **Question:** Does the user find the answer immediately?
- **Audit Check:** Look at the first scroll (Above the Fold). Is the answer there? Or is it fluff?
- **Competitor Check:** Do winners answer the question in the H1/Subtitle?

### 2. The Information Gain Score

Google patents cite "Information Gain" – adding new value to the corpus.

- **Audit Check:** Does the client's content add new data, a new perspective, or a new synthesis? Or is it just a rewrite?
- **Agentic Task:** Compare semantic distance between client content and top result. If >90% similar = Low Information Gain.

### 3. Entity Density & Co-occurrence

LLMs understand topics via Entities (People, Places, Things, Concepts).

- **Audit Check:** Extract entities from Top 3 results. Compare to client.
- **Example:** Top results for "CRM" mention "Salesforce," "HubSpot," "Automation," "Pipeline." If client misses "Pipeline," they lack topical relevance.

### 4. The "Multimedia" Advantage

Video is the #1 "Trust Signal" for AI and Google.

- **Audit Check:** Do competitors have embedded video?
- **Gap Logic:** If they do and you don't, you lose. If they don't and you add one, you win.

---

## KB-HEADER-STRUCTURE-OPTIMIZATION-v1.0

### 1. The "Skeleton" of Ranking

Headers (H1-H6) are the skeleton AI agents read to understand the document.

**Rule:**
- H1 must match User Intent
- H2s must be the "Pillars" of the answer
- H3s must be the "Details"

### 2. Pattern Matching the SERP

If 8/10 results have an H2 for "Pricing," you must have an H2 for "Pricing." It is a user expectation.

**Agentic Task:** Map the H2 frequency across the Top 10. List the "Mandatory Headers."

### 3. The "FAQ" Schema Opportunity

FAQs are the easiest way to capture Long-Tail queries and Voice Search.

- **Audit Action:** Scrape "People Also Ask" (PAA) for the keyword
- **Optimization:** Ensure the article has an H2 "Frequently Asked Questions" that answers the top 3 PAA queries directly

---

## KB-CONTENT-GAP-TO-BRIEF-PIPELINE-v1.0

### 1. From Analysis to Instruction

An audit without a brief is just noise.

**The Bridge:** The audit identifies *what* is missing. The brief dictates *how* to build it.

### 2. The "Superior Brief" Template

```yaml
brief:
  target_keyword: "[Primary Keyword]"
  user_intent: "[Informational/Transactional]"
  winning_structure: "[H2 List based on Competitors]"
  the_10x_element: "[Unique Data/Video/Tool]"

```

### 3. Module 3 Handoff

- Output the brief in Markdown
- Module 3 (Writer) ingests Markdown to generate the draft
- This ensures the "Audit" directly controls the "Creation"

## KB-AGENTIC-AUDIT-ARCHITECTURE-v1.0

### 1. The Tech Stack

- **n8n:** The orchestrator. Triggers the workflow (e.g., "New Client Onboarding")
- **Firecrawl:** The eyes. Scrapes the web cleanly into Markdown
- **OpenAI/Claude API:** The brain. Analyzes the text
- **Airtable/Google Sheets:** The memory. Stores the findings

### 2. Workflow Automation

1. **Trigger:** User inputs Domain + Competitors
2. **Action 1:** Firecrawl scrapes Client Sitemap + Competitor Sitemaps
3. **Action 2:** LLM compares Topic Clusters
4. **Action 3:** LLM identifies "Missing Clusters" (Niche Interest Trees)
5. **Action 4:** LLM writes the Executive Summary
6. **Output:** PDF Report generated

### 3. "Self-Healing" Audits

Agents can not only find errors but fix them.

- **Example:** Agent finds missing Meta Description → Agent writes new Description → Agent pushes to CMS (via API)
- **Note:** Module 7 focuses on identifying the fix; Module 6 executes the fix
