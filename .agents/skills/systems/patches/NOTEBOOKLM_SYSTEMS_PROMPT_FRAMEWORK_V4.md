# NotebookLM Systems Prompt Framework V4

## Prompt 1: Simplify The Knowledge Engine
Given the current ULTRAMIND knowledge engine document, extract the durable architecture and remove manifesto, comparison noise, and speculative excess. Return a cleaner machine-friendly version.

## Prompt 2: COSMS Object Design
Design the minimum viable COSMS object model for agent-upgradeable private knowledge. Return node fields, edge fields, cluster logic, and synthesis log structure.

## Prompt 3: Neo4j Readiness
Map the proposed schema to a future Neo4j model without overcommitting the live runtime. Return safe graph mappings and what should stay derived.

## Prompt 4: Flowith-Like Front End
Propose a simple card-first interface with optional graph view, sortable clusters, self-association suggestions, and low cognitive load.

## Prompt 5: Gap Query Extraction
Identify what the current document still does not specify clearly. Return exact gap queries for NotebookLM to answer next.

## Output Normalization
Normalize every run into:
- durable_principles
- schema_fields
- relation_types
- ui_contract
- graph_mapping
- gap_queries
- patch_recommendations
