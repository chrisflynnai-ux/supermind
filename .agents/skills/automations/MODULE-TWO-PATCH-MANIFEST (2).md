### PATCH: TEMPLATE_COORDINATOR
**Priority:** Critical  
**Source:** Nick Puru, Kevin Badi

**Template:**
```python
"""
[AUTOMATION_NAME] Coordinator
Generated by MODULE 2: Automation Builder
Implements: PCE Framework + Self-Annealing Loop
"""

import subprocess
import json
import os
from datetime import datetime
from pathlib import Path

class AutomationCoordinator:
    """
    The "Manager" layer in PCE Framework.
    Reads Plans, executes Scripts, handles Self-Healing.
    """
    
    def __init__(self, sop_path: str):
        self.sop_path = sop_path
        self.run_id = f"run_{datetime.now().strftime('%Y%m%d_%H%M%S')}"
        self.max_retries = 3
    
    def run_with_annealing(self, script_path: str, inputs: dict) -> dict:
        """
        Executes a script. If it fails, triggers self-healing loop.
        
        PCE Execution Layer: Scripts are "dumb but reliable."
        Self-Annealing: Failures train the system.
        """
        attempt = 0
        
        while attempt < self.max_retries:
            try:
                # 1. Execute Script (Execution Layer)
                result = subprocess.run(
                    ['python', script_path, json.dumps(inputs)],
                    capture_output=True,
                    text=True,
                    timeout=300
                )
                
                # 2. Check for Failures
                if result.returncode != 0:
                    raise ExecutionError(result.stderr)
                
                # 3. Parse Sandbox-Filtered Output
                return json.loads(result.stdout)
                
            except Exception as e:
                # 4. Self-Annealing Hook
                print(f"❌ Attempt {attempt + 1} failed: {e}")
                print("🔧 Engaging Auto-Fixer...")
                
                # Update Information Layer (Memory)
                self._update_sop_constraints(script_path, str(e))
                
                # Patch Implementation Layer (Code) - Flag for review
                self._log_fix_needed(script_path, str(e))
                
                attempt += 1
        
        raise MaxRetriesExceeded(f"{script_path} failed after {self.max_retries} attempts")
    
    def _update_sop_constraints(self, script_name: str, error: str):
        """
        Append learned constraint to SOP's Dynamic Memory section.
        This is the Information Layer update in Double-II.
        """
        constraint_entry = f"| {datetime.now().strftime('%Y-%m-%d')} | {type(error).__name__} | {error[:100]} |\n"
        
        with open(self.sop_path, 'a') as f:
            f.write(constraint_entry)
        
        print(f"📝 Updated SOP with learned constraint")
    
    def _log_fix_needed(self, script_path: str, error: str):
        """Log that Implementation needs review."""
        log_entry = {
            "timestamp": datetime.now().isoformat(),
            "script": script_path,
            "error": error,
            "status": "needs_review"
        }
        
        log_path = Path("logs/annealing_log.json")
        log_path.parent.mkdir(exist_ok=True)
        
        existing = json.loads(log_path.read_text()) if log_path.exists() else []
        existing.append(log_entry)
        log_path.write_text(json.dumps(existing, indent=2))
    
    def execute_workflow(self, initial_input: dict) -> dict:
        """
        Main workflow execution following PCE pattern.
        
        1. Planning: Load SOP
        2. Coordination: This method (branching, sequencing)
        3. Execution: run_with_annealing() calls
        """
        print(f"🚀 Starting workflow: {self.run_id}")
        
        # Step 1: [STEP_NAME]
        result_1 = self.run_with_annealing(
            "scripts/step_1.py",
            initial_input
        )
        
        # Step 2: [STEP_NAME]
        result_2 = self.run_with_annealing(
            "scripts/step_2.py",
            result_1
        )
        
        # Final: Sandbox-filtered summary
        return {
            "run_id": self.run_id,
            "status": "success",
            "summary": f"Processed {result_2.get('count', 'N/A')} items",
            "full_results_ref": f"outputs/{self.run_id}.json"
        }


class ExecutionError(Exception):
    """Script execution failed."""
    pass

class MaxRetriesExceeded(Exception):
    """Self-healing exhausted all retries."""
    pass


if __name__ == "__main__":
    coordinator = AutomationCoordinator("planning/SOP.md")
    result = coordinator.execute_workflow({"source": "input"})
    print(json.dumps(result, indent=2))
```

### PATCH: ERROR_QUARANTINE
**Priority:** High  
**Source:** Dr. Maryam Miradi

**Pattern:**
```yaml
context_quarantine:
  definition: "Isolate heavy processing to prevent context rot"
  
  trigger: "Complex task (e.g., analyzing 50 PDF documents)"
  
  steps:
    spawn_sub_agent:
      action: "Spin up temporary helper agent"
      context: "Fresh, minimal context"
    
    execute:
      action: "Helper runs heavy tools/scripts"
      data: "Full documents processed in isolation"
    
    summarize:
      action: "Helper returns only the answer"
      format: "Sandbox-filtered summary"
    
    kill:
      action: "Helper agent destroyed"
      result: "Bloated context dies with it"
  
  benefit: "Main coordinator context stays lean"
  
  code_pattern: |
    def quarantined_task(task_func, inputs: dict) -> dict:
        """
        Run heavy task in isolated subprocess.
        Only summary returns to main context.
        """
        # Spawn subprocess (fresh Python process = fresh context)
        result = subprocess.run(
            ['python', '-c', f'''
import json
from heavy_task import {task_func.__name__}
result = {task_func.__name__}({json.dumps(inputs)})
# Return only summary
print(json.dumps({{"summary": result["summary"], "ref": result["ref"]}}))
            '''],
            capture_output=True,
            text=True
        )
        
        return json.loads(result.stdout)
```

## STRATEGIC SUMMARY

### Architecture Upgrades
1. **Double-II Enforced** — Every generated package has Information + Implementation layers
2. **PCE Framework** — Plan (SOP) → Coordinate (Manager) → Execute (Scripts)
3. **Sandbox Filtering** — Scripts return summaries, store full data externally

### Template Upgrades
1. **Dynamic Memory SOP** — Auto-updated "Learned Constraints" section
2. **Self-Healing Coordinator** — Try/catch/fix/retry loop built-in
3. **Sandbox Scripts** — Heavy processing isolated, compressed output only

### Business Pattern Upgrades
1. **Lead Gen** — Apify discovery → Enrichment → Verification → Circuit breaker
2. **Outreach** — Check status → Personalize locally → Inject to campaign
3. **Content** — Extract → Framework format → Save for review (no auto-post)

### Resilience Upgrades
1. **Recursive Annealing** — Failures train the system
2. **Context Quarantine** — Heavy tasks in sub-processes

### Expansion Upgrades
1. **Visual Bridge** — Mermaid diagrams from code
2. **Deployment Scripts** — Modal/serverless configs generated
