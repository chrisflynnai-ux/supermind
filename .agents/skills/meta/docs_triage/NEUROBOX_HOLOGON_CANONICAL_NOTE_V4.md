# NeuroBox Hologon Canonical Note V4

Status: active_reference
Purpose: Correct recurring drift in Neuro-Box / Hologon explanations across planning and skill docs.

## Canonical Model
This is a relational array you call a Hologon.
It is functionally an 8-dimensional object displayed in 3D:
- 6 outer sides
- 1 theoretical center with two aspects
- 1 mini-axis within the center

## Canonical Side Mapping
- Bottom: `BODY` -> SAFE -> Physical -> Dopamine -> Status
- Top: `HEART` -> SPECIAL -> Emotional Stimulation -> GABA
- Left: `MIND` -> SMART -> Mental -> Acetylcholine -> Stress Relief
- Right: `SPIRIT` -> SIGNIFICANCE -> Spiritual -> Serotonin -> What Others Want
- Front: `PSYCH` -> SUPERIOR -> Ego/Obsession -> Adrenaline (ego wants to feel superior, found the best solution)
- Back: `CONSCIENCE` -> SUPPORTED -> Social Awareness -> Oxytocin (wants connection, community, values, identity support)

## Center
The center is `SOUL`.
Two recurring commercial expressions:
- `SALVATION` -> Whole solution transformation, transcend past struggles, absolution from guilt
- `STEAL-of-a-Deal` -> Irresistible value offer (Adrenaline + Dopamine: anticipation, excitement, value stacking, bonuses)

## 7F Functions (Business Language)
1. FILTER → Establish safety (BODY)
2. FRAME → Create uniqueness (HEART)
3. FOCUS → Deliver clarity (MIND)
4. FORM → Provide proof (SPIRIT)
5. FIXATE → Enable action (PSYCH)
6. FORGE → Build belonging (CONSCIENCE)
7. FLOW → Integrate transformation (SOUL)

## Simplified Alternative
When the full Hologon is too heavy, use a simple 4-step diagram with two parts in each step.
Do not let the simplified view overwrite the canonical mapping above.

## Correction Rule
If a document swaps BODY/HEART, MIND/SPIRIT, PSYCH/CONSCIENCE, or the related neurochemicals, patch it.
If a document keeps the overall intent but uses older 3-phase or misordered explanatory language, patch rather than discard.
