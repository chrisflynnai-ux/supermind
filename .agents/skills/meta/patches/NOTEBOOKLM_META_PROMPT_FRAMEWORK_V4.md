# NotebookLM Meta Prompt Framework V4

Use these prompt frames to generate enhancement inputs before applying any meta-family upgrade.

## Prompt 1: Modern Skill Architecture
Analyze the attached meta skill source and identify the strongest modern patterns for Claude and multi-model agent skill design. Return: structural upgrades, discovery/frontmatter upgrades, decomposition opportunities, failure modes, and routing implications.

## Prompt 2: Pipeline Contradiction Check
Compare the source meta document against the canonical atomic pipeline `Skill Atoms -> NotebookLM Router -> Claude Synthesizer`. Return: stale doctrine, Prism-era conflicts, and exact instructions that should be retired or rewritten.

## Prompt 3: Relational Operations Design
Given the meta family roles `meta_zpwo`, `meta_mma`, `meta_skill_builder`, `meta_skill_architect`, `meta_skill_auditor`, and `meta_template_system`, propose explicit upstream/downstream contracts, wake conditions, review ownership, and patch ownership.

## Prompt 4: Refactor Readiness
Assess whether the source should be patched, decomposed, rebuilt, or archived. Return: verdict, rationale, missing atoms, and recommended V4 target shape.

## Output Normalization
Normalize each NotebookLM run into:
- heuristics
- failure_modes
- missing_atoms
- stale_doctrine
- structural_upgrades
- routing_notes
- open_questions
